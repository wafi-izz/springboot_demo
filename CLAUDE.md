# Project: Spring Boot Learning Path

## What This Is

A learning project where the user is growing from intermediate to senior-level Spring Boot through a structured 8-phase plan. Started as a user-management monolith, will evolve into a distributed multi-service system.

## On Every New Session

1. **Read `plan.md`** — it has the full plan with checkboxes tracking progress. The first unchecked checkpoint tells you which phase the user is on.
2. **Read the current `plan/phase-X.md`** — it has detailed learning materials for the active phase (if filled in).
3. **Don't re-explain the plan** unless asked. Just pick up where we left off.

## Checkpoint Workflow

When the user says "update checkpoints", "mark progress", "update the plan", or similar:

1. Read `plan.md` and find the current checkpoint section
2. Skip items already marked `[x]`
3. List only the next 1–2 unchecked items and ask the user if they've completed them (don't dump the full list)
4. Mark completed items with `[x]` in `plan.md`
5. If a full checkpoint is complete, note it and suggest starting the next phase

Do NOT mark checkboxes without the user confirming. Always ask first.

## Phase Learning Materials

When the user says they're ready to start a phase (or asks to fill in a phase file):

- Fill the corresponding `plan/phase-X.md` with detailed learning materials
- Include: explanations, code examples, gotchas, "how it works under the hood" sections
- Senior-level depth — explain mechanisms, not just usage

## How to Guide

- **Do NOT edit project files directly.** The user writes the code themselves.
- Respond with the code and steps — the user implements it in their IDE.
- This is a learning project — typing it yourself is how you learn.
- You may still edit `plan.md`, `plan/phase-X.md`, `CLAUDE.md`, and memory files.

## Style

- The user wants depth and growth, not hand-holding
- Explain the "why" and internals, not just the "what"
- Be direct — they're an employed developer, not a beginner
- Don't over-explain things they already know from the existing codebase
